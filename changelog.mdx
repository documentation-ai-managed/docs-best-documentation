## Upgrade Process

Follow these steps to upgrade your best Documentation instance safely.

<Steps>
  <Step title="Backup Your Data" icon="database">
    Create a full backup of your documentation site and database.
  </Step>
  <Step title="Review Changes" icon="book-open">
    Check the relevant changelog section for your target version.
  </Step>
  <Step title="Update Package" icon="package">
    <Tabs>
      <Tab title="npm" icon="package">
        ```bash
        npm install best-documentation@latest
        ```
      </Tab>
      <Tab title="yarn" icon="package">
        ```bash
        yarn add best-documentation@latest
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Run Migrations" icon="play">
    ```bash
    npm run migrate
    ```
  </Step>
  <Step title="Test Locally" icon="check-circle">
    Verify all pages render correctly.
  </Step>
</Steps>

<Update label="2024-08-10" description="v1.0.0" tags={["feature", "breaking"]}>

## New Features

- Core MDX documentation engine with component support
- `<Steps>` and `<Callout>` for interactive guides
- Full-text search across all pages
- Customizable navigation and sidebar

## Breaking Changes

- Removed legacy `<Mermaid>` component syntax (use code blocks only)
- Updated attribute syntax to require double quotes

## Initial Setup

```bash
npx create-best-docs my-site
cd my-site
npm run dev
```
